# Streamlit-приложение для анализа тональности отзывов

## Стек проекта

- Язык и платформа: `Python 3.14`
- Интерфейс: `Streamlit`
- Обработка данных: стандартная библиотека Python (`json`, `csv`, `re`, `collections`, `typing`)
- ML-стек: `Hugging Face Transformers` + `PyTorch`
- Модель анализа тональности: `blanchefort/rubert-base-cased-sentiment`
- Модель для zero-shot классификации эмоций и тем: `MoritzLaurer/mDeBERTa-v3-base-mnli-xnli`
- Поддерживаемые форматы входных данных: `.txt`, `.md`, `.csv`, `.json`
- Внутренний формат обработки: единый JSON-подобный документ с текстом и метаданными

## Заметки по проекту

- Основной файл приложения: `emotion.py`
- Приложение написано в одном файле
- Поддерживается ручной ввод текста
- Поддерживается загрузка одного или нескольких файлов `.txt`, `.md`, `.csv`, `.json`
- Если загружены файлы, для анализа используется текст из файлов, а не из поля ввода
- Все источники сначала нормализуются во внутренний единый JSON-подобный документ
- В интерфейсе показываются тональность, эмоции, темы, важные фрагменты и сводный анализ по всем отзывам

## Возможности

- Анализ общей тональности текста
- Определение эмоций
- Определение тем отзыва
- Подсветка важных фрагментов
- Сводный анализ всех отзывов
- Шкала общей эмоциональности от негативной к позитивной
- Статистика по количеству негативных, нейтральных и позитивных отзывов
- Выделение повторяющихся замечаний и жалоб

## Формат входных данных

Чтобы приложение корректно разделяло отзывы внутри одного файла, используйте один из форматов ниже:

### TXT / MD

Один отзыв на блок. Рекомендуемый разделитель между отзывами:

```text
Очень вкусный кофе и приятный персонал.
---
Столы были грязные, а заказ несли слишком долго.
---
Нормальное место, без ярких впечатлений.
```

Допустим и вариант с пустой строкой между отзывами.

### CSV

Один отзыв в одной строке. Рекомендуемая колонка: `review`.

```csv
review,rating
"Очень вкусный кофе и уютная атмосфера",5
"Грязные столы и долгое обслуживание",2
"Обычное место, без особых эмоций",3
```

Также поддерживаются названия колонок `text`, `comment`, `content`, `message`, `feedback`, `body`.

### JSON

Рекомендуемый формат: массив отзывов или объект с полем `reviews`.

```json
{
  "reviews": [
    { "review": "Очень вкусный кофе и доброжелательный персонал" },
    { "review": "Столы были грязные, обслуживание медленное" },
    { "review": "Обычное место, все нейтрально" }
  ]
}
```

Также поддерживается список объектов:

```json
[
  { "text": "Очень вкусный кофе и доброжелательный персонал" },
  { "text": "Столы были грязные, обслуживание медленное" }
]
```

## Локальный запуск

Установка зависимостей:

```bash
"C:\Users\user\AppData\Local\Programs\Python\Python314\python.exe" -m pip install streamlit transformers torch
```

Запуск приложения:

```bash
"C:\Users\user\AppData\Local\Programs\Python\Python314\python.exe" -m streamlit run emotion.py
```

Быстрый запуск через bat-файл:

```bash
run_emotion.bat
```
